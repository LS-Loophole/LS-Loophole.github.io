UCMS v1.5.0 XSS dir parameter

Vulnerability Type :

Cross Site Scripting(XSS)

Vulnerability Version :

1.5.0

Recurring environment:

  • Windows 10
  • PHP 5.4.5
  • Apache 2.4.23

Vulnerability Description AND recurrence:

Vulnerability recurrence:

Vulnerability URL:
www.ucm1.5.0.com/ucms/index.php?do=sadmin_fileedit&dir=/&file=index.php

There is an XSS vulnerability in the dir parameter. You can splice <script>alert('CNVD')</script> here to trigger the vulnerability

www.ucm1.5.0.com/ucms/index.php?do=sadmin_fileedit&dir=/<script>alert('CNVD')</script>&file=index.php

Vulnerability analysis:

Because this cms uses routing, so?do=sadmin_fileedit&dir=/&file=index.php
The actual operation of \ucms\sadmin\fileedit.php

There are three trigger parameters here, namely: do=sadmin_fileedit, dir=/, file=index.php.

Look here for the vulnerability trigger points in the source file \ucms\sadmin\fileedit.php

The $dir variable at line 87 has an xss vulnerability

Go back here to find out why there is a problem with $dir here! Start from the beginning

The first few lines are the previous conditions
Line 2, this file can not be accessed directly. Need to visit on behalf of the route through the middle.
Line 3, judge the current level. After logging in here, it is 3 so it is not executed
Line 4, determines whether there is a constant AdminFileedit. It exists by default here so it is not executed

Line 7, need to GET parameters passed in a dir, or exit PHP
Line 8, if the incoming transmission parameter dir is null, a default assignment '/'
Line 11, $_GET [ 'dir'] $ incoming variable GETDIR
Line 12-16, the $ _GET [ 'dir'] of data into variable $dir
Line 17, join the current webpage directory $_GET['dir'] and'/' and then pass it into $alldir
Line 18, prevent directory traversal loopholes. When the dir we passed in
contains two… After outputting'error dir' and exiting PHP

Line 22, we need to pass GET variables file. Otherwise, exit PHP after outputting'no file'
Line 23, the incoming file $ filename
Line24, prevent directory traversal loopholes. If the variable exists… then exit after outputting'error filename'
Line 25, the function call isedit 50 rows. The isedit function uses the foreach function to return only the suffix of $filename. Returns true if the suffix is one of the $array. If it is satisfied here, the content of line 26 will not be executed
Line 31, where the vulnerability is triggered no incoming POST, so here we do not need to look at the code here
Line 44, to determine $ alldir. $ Filename spliced out whether a file exists. If there is no $content assignment is empty, if it exists, the content html inside will be instantiated and assigned to the $content variable

Summarize here:

  1. The role of do is to specify the back-end file of the incoming data
  2. The dir variable needs to be passed in without filtering
  3. The file variable needs to be passed in a random file name and the suffix
    is any one of'php','css','js','htm','html','txt'.

Then the vulnerability point is directly output $dir without any filtering, so the xss vulnerability is caused
