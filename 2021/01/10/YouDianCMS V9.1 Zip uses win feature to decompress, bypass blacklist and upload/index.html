Zip uses win feature to decompress, bypass blacklist and upload

Vulnerability Type :

File upload

Vulnerability Version :

YouDianCMS V9.1

Recurring environment:

  • Windows 10
  • PHP 5.4.5
  • Apache 2.4.23

Vulnerability Description AND recurrence:

Vulnerability hazard

Through this vulnerability, you can upload zip compressed package and use specific methods to bypass the blacklist restriction of zip resolvable suffix by using Windows features. In order to execute the command, to achieve the purpose of controlling the server!

Vulnerability analysis

The vulnerabilities are in the following areas: youdiancms, app, lib and action\ AdminBaseAction.class.php

As you can see, line 116 clearly says that if the following occurs, the upload will fail

$map = array('php'=>true, 'jsp'=>true, 'asp'=>true, 'aspx'=>true);

This is quite different from the previous version. As you can see from the previous version, there is no such security detection at all!

Therefore, in the latest version of v9.1, the 9.0 compressed package upload method has failed, because it will detect the suffix of each file in the packet 

if(isset($map[$ext])){
    $this->ajaxReturn(null, "模板不能包含{$ext}文件" , 0);
}

The overall logic of this paragraph is as follows

Loop out all the contents in the zip package, and $count is the collection of the current directory

for($n = 0; $n < $count; $n++){

If the folder attribute of the current file is not empty, the next judgment will be performed

 if( $list[$n]['folder']) continue;

Put the name of the current file into $file name!

$filename = strtolower($list[$n]['filename']);

Determine whether the current file is common_ en.php Or common_ cn.php And if so, no operation is performed. Continue down

    if(stripos($filename, '/common_en.php') || stripos($filename, '/common_cn.php') ){
}else{

Using YD_ file_ Ext returns the suffix of the current file name and converts all characters to lowercase with the strtolower() function

$ext = strtolower(yd_file_ext($filename));

If the suffix in $ext exists in the field $map [array (‘php ‘= > true,’ JSP ‘= > true,’ ASP ‘= > true,’ ASPX ‘= > true);]

if(isset($map[$ext])){

That is, if the suffix is PHP or JSP or ASP or ASPX, execute the following command. Return character, and upload failed!

$this->ajaxReturn(null, "模板不能包含{$ext}文件" , 0);

But there is a problem! It does not remove the spaces on both sides of the string! Here comes a question!We can use the white space feature of windows to bypass!
